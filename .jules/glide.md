## 2025-03-02 - Implementing Reduced Motion in Framer Motion + Astro
**Learning:** When using Framer Motion inside Astro React Islands, checking system preferences for reduced motion is essential for accessibility. While `useReducedMotion()` from `motion/react` is excellent for conditionally stripping animation parameters (like setting `duration: 0.01` or removing vertical `y` translations entirely) from complex React elements like the `WeatherHero.tsx`, we still need a global CSS fallback (`@media (prefers-reduced-motion: reduce)`) in a root stylesheet (like `src/index.css`) to catch any CSS-only interactions or Astro/static UI elements. Combining both gives total coverage without JS overhead on static assets.
**Action:** Always complement React/Framer motion optimizations with a global CSS `@media (prefers-reduced-motion: reduce)` block setting `animation-duration: 0.01ms !important; transition-duration: 0.01ms !important;` to ensure holistic accessibility.
